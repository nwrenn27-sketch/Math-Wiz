Building It ‚Äî Challenges

What was hard to build

01
Rendering math in the browser
Calculus requires LaTeX ‚Äî symbols like ‚à´, ‚àÇ, ‚àë. Integrating MathJax and making it render dynamically inside JavaScript-injected DOM elements (not just static HTML) took significant trial and error.
02
Fuzzy answer checking
Students write answers in dozens of formats: "65 m/hr", "65", "$65$", "‚âà65". Building a normalizer that accepts all valid forms without false positives required layered regex and numeric comparison logic.
03
Drawing diagrams with Canvas API
Each problem type needs a unique visual ‚Äî right triangles, function graphs, 3D cubes, balloons. Every diagram is drawn in code from scratch using the HTML5 Canvas API with no external libraries.

How the step reveal logic works

// Called every time the user clicks "Show Next Step"
function showNextStep() {
  const problem = appState.currentProblem;
  if (!problem) return;

  const nextIndex = appState.currentStepIndex + 1;

  if (nextIndex >= problem.steps.length) {
    revealAnswer();  // all steps done ‚Äî show the final answer
    return;
  }

  appState.currentStepIndex = nextIndex;
  appState.stepsRevealed.push(nextIndex);

  renderStep(problem.steps[nextIndex], nextIndex);
  updateProgress();
  updateButtonStates();

  // Re-highlight diagram elements specific to this step
  if (problem.steps[nextIndex].highlightElements) {
    highlightDiagramElements(problem.steps[nextIndex].highlightElements);
  }
}
üí° Key insight: Each step is an object in an array. The function tracks an index into that array ‚Äî when the index exceeds the array length, the answer is automatically revealed. Steps can also carry highlightElements to update the diagram in sync with the explanation.
Under the Hood ‚Äî Answer Checking

Fuzzy answer matching

practice-app.js 
function checkAnswer(studentAnswer, correctAnswer) {

  const normalize = (str) => str
    .toLowerCase()
    .replace(/\s+/g, '')                        // "65 m/hr" ‚Üí "65m/hr"
    .replace(/[\\${}]/g, '')                     // strip LaTeX chars
    .replace(/\\frac\{([^}]+)\}\{([^}]+)\}/g, '($1/$2)') // \frac{1}{2} ‚Üí (1/2)
    .replace(/‚âà|~|approximately/g, '');          // ignore ‚âà symbol

  const a = normalize(studentAnswer);
  const b = normalize(correctAnswer);

  if (a === b) return true;   // exact match after normalization

  // Extract the numeric part and compare with tolerance
  const numA = parseFloat(a.match(/[\d.]+/));
  const numB = parseFloat(b.match(/[\d.]+/));

  if (numA && numB) return Math.abs(numA - numB) < 0.01;

  return a.includes(b) || b.includes(a);  // substring match fallback
}
üí° Key insight: Three layers of matching ‚Äî (1) exact match after stripping formatting, (2) numeric tolerance of ¬±0.01 for floating point answers, (3) substring containment as a final fallback. This lets "65", "65 m/hr", "$65$", and "‚âà65" all be accepted for the same correct answer.
